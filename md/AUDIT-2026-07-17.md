# Audit — 2026-07-17

A comparison of ARCHITECTURE.md against what the code actually runs, plus a
feature-by-feature review of everything added over the project's history:
is it used, does it help, or does it just exist. Ends with the actions taken.

## Headline

ARCHITECTURE.md was *accurate* — everything it described existed in code — but
a large share of what it documented was not on any live path, and the biggest
problem wasn't in the doc at all: **six draft PRs (#6–#11) had never been
merged**, including the ones that fix the known blockers. Main was still
running the pre-audit system: the daily paper-run CI failed every day
(July 14–16, dying at the same auth step), the forward record was stalled at
**2026-07-13**, and the keyless Yahoo data source that fixes both sat unmerged
in PR #9.

## Documented but not used

- **The ICIR machinery — `factor/`, `search/`, `gate/`** (§2 + Loops A/B of
  ARCHITECTURE.md). No production code imports any of it; they are standalone
  research CLIs. They have never selected a strategy: the real-data verdict was
  `viable: 0` for every strategy, and the shipped config
  (`mean_reversion, params: {}`) is the code default, not a search output.
  The July-16 refocus standardized on trade-level grading (scorecard, failure
  buckets, robust Sharpe) instead. PR #8 already reframed §2 as "candidate
  generation, not the judge." **Kept as research tools; not the grading path.**
- **`pairs` strategy.** Its forward track was deleted as a dead one-day record;
  nothing ran it. **Removed in this cleanup** (strategy, runner/forward/compare
  branches, `pairs_target_orders`, tests).
- **`BucketFilter` and `WinProbGate` overlays** (+ the IRLS logit helpers).
  Built for the loss-learning bake-off and lost it — only ConvictionGate is on
  the forward path; WinProbGate was inert at defaults. **Removed in this
  cleanup**; they live in git history.
- **`evaluate_robust.py`.** Only the dashboard's bake-off panel calls it. It
  answered its question (conviction lifts point Sharpe ~5× but nothing clears
  the bootstrap noise band at this data scale). Kept — it is the robust judge
  for any future variant.
- **The world model (§5).** Correctly documented as "seams only, nothing
  built." Frozen per the refocus; nothing to do.
- **`journal/forward/agent/`** — a stale one-day track (2026-07-09, return 0.0,
  anchored on the old NVDA/SPY config). Removed from `paper-state` so the next
  agent tick re-anchors on the current 65-name universe.

## Actions taken (2026-07-17)

1. **Merged PR #7** (FINDINGS.md audit), **PR #8** (refocus fixes), and
   **PR #9** (Yahoo keyless data source; resolved its conflict with #8 in
   favor of the keyless design and re-enabled the CI schedule). The forward
   record can now grow unattended, keyless.
2. **Fixed a CI landmine introduced by #8+#9 together:** `paper_cron.sh` ticks
   the agent engine daily, which needs `NVIDIA_API_KEY` — not present in the
   workflow env. Under `set -e` that would have crashed every freshly-fixed
   run. The agent tick is now skipped when the key is unset and non-fatal when
   it fails; the workflow passes `NVIDIA_API_KEY` through from repo secrets,
   so adding that secret enables the agent's forward record.
3. **Deleted the `pairs` strategy** and all its special-case branches
   (`runner.py`, `forward.py`, `compare.py`, `strategy_runner.py`) and tests.
4. **Deleted the losing overlays** (`BucketFilter`, `WinProbGate`, logit
   helpers) and their tests; `--overlay` now accepts `none|conviction`.
5. **Trimmed ARCHITECTURE.md and README** to match (pairs row, overlay list).
6. **Removed the stale `journal/forward/agent/` track from `paper-state`** so
   the next agent tick anchors fresh on the current config.
7. Wrote this audit.
